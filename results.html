GLISSANDO 3 Installation and Setup

This section describes the installation and basic usage of GLISSANDO 3 for Monte Carlo Glauber simulations. All simulations presented here were performed on a Linux environment using ROOT for analysis.

Prerequisites

  • Linux environment (Ubuntu recommended)
  • g++ compiler with C++11 support
  • ROOT (version 5.32 or 6.18) for compilation and analysis
  • Doxygen (version 1.74 or newer) for documentation generation
  • texlive-fonts-recommended for LaTeX-style PDFs

Download and Extract


# Download the package (replace xxx with actual version)
wget https://www.ujk.edu.pl/homepages/mryb/GLISSANDO/glissando3xxx.tar.gz

# Extract the archive
tar -zxvf glissando3xxx.tar.gz
cd glissando3

Compile the Code

make

Run a Test Simulation


# Edit input configuration
vim input/input.dat

# Run GLISSANDO
./glissando3

# Output ROOT file
output/glissando.root

Documentation and Demo Runs


make cleandoc
make doc

./demo_ver_2.sh
./demo_ver_3.sh
./centrality.sh

Batch Mode Execution


./demo_ver_2.sh < one.dat
./demo_ver_3.sh < one.dat

Cleanup and Packaging

  • make clean – remove object and binary files
  • make cleandoc – remove documentation
  • make cleanoutput – remove output ROOT files
  • make package – create distributable archive

Collision Systems Studied at √sNN = 5.36 TeV

GLISSANDO 3 simulations were performed at LHC energies for a variety of collision systems, ranging from the smallest to the heaviest nuclear configurations. This allows a systematic study of system-size dependence of initial-state geometry.

A. Proton–Proton (p+p) Collisions

  • NUMA = 1, NUMB = 1
  • Number of events: 100,000
  • Partonic substructure: 3 constituent quarks per nucleon

Key observables:

  • Mean number of wounded sources (Nw)
  • Eccentricity ε2 distributions
  • Participant-plane source projections

Even in the smallest collision system, geometric fluctuations generate non-zero eccentricities, highlighting the importance of sub-nucleonic structure.

B. Oxygen–Oxygen (O+O) Collisions

  • NUMA = 16, NUMB = 16
  • Number of events: 100,000

O+O collisions represent an intermediate system where collective behavior begins to emerge while fluctuations remain significant. The transition from small to medium-sized systems is clearly visible in the distributions of Nw and εn.

Results include:

  • Nw and Ncoll distributions
  • Centrality-dependent eccentricities
  • Transverse density profiles

C. Xenon–Xenon (Xe+Xe) Collisions

  • NUMA = 129, NUMB = 129
  • Deformed nuclear geometry

Xe+Xe collisions introduce intrinsic nuclear deformation effects. Event-by-event fluctuations in geometry lead to enhanced higher-order eccentricities compared to spherical nuclei.

These features provide a sensitive test of Glauber modeling and are directly relevant for interpreting LHC Xe+Xe data.

D. Lead–Lead (Pb+Pb) Collisions

  • NUMA = 208, NUMB = 208
  • Largest system studied

Pb+Pb collisions exhibit the strongest collective effects and the smoothest average geometry. The large number of participants suppresses relative fluctuations, making this system ideal for studying bulk QGP properties.

Key results:

  • Centrality classes based on Nw
  • Average eccentricities ε2, ε3
  • Two-dimensional participant density projections

Summary of Results

The GLISSANDO 3 simulations demonstrate a clear system-size dependence of initial-state geometry. From p+p to Pb+Pb collisions, the increase in participant number leads to a smoother geometry and reduced relative fluctuations, while intrinsic nuclear structure plays a significant role in intermediate systems.

These results form the foundation for subsequent hydrodynamic modeling and comparison with experimental measurements at the LHC.
